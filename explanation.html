QA Agent - Complete Code Explanation

This document explains every line of code in the QA Agent project. The project is a local LLM-powered chatbot with tool calling, an evaluation system, and a web dashboard. All code runs entirely on your computer using the Llama-3 model.

Overview

The project consists of 4 main Python files:

  • setup_local.py - Downloads the Llama-3 model from Hugging Face
  • agent.py - Core chatbot logic with tool calling
  • server.py - FastAPI REST API server
  • evaluation.py - LLM-as-judge evaluation system

The architecture flow is:

  1. User sends a message via the web dashboard
  2. Dashboard calls the /chat API endpoint
  3. Server passes the message to the agent
  4. Agent uses Llama-3 to generate a response (possibly calling tools)
  5. Response is returned to the dashboard

Chapter 1: Model Setup (setup_local.py)

This file downloads the Llama-3 model from Hugging Face Hub. We use a quantized (compressed) version of the model called Q4_K_M which is about 2GB and runs efficiently on CPU.

setup_local.py - Complete File
import os
from huggingface_hub import hf_hub_download

def download_model():
    model_name = "bartowski/Llama-3.2-3B-Instruct-GGUF"
    filename = "Llama-3.2-3B-Instruct-Q4_K_M.gguf"
    
    print(f"Downloading {filename} from {model_name}...")
    
    # Create models directory if it doesn't exist
    if not os.path.exists("models"):
        os.makedirs("models")
        
    try:
        model_path = hf_hub_download(
            repo_id=model_name,
            filename=filename,
            local_dir="models",
            local_dir_use_symlinks=False
        )
        print(f"Successfully downloaded model to: {model_path}")
        return model_path
    except Exception as e:
        print(f"Error downloading model: {e}")
        return None

if __name__ == "__main__":
    download_model()

  • Line 1-2: Import os for file operations and hf_hub_download from the Hugging Face Hub library to download models.
  • Line 4-6: Define the function and specify which model to download. We use bartowski/Llama-3.2-3B-Instruct-GGUF which is a community-quantized version of Meta's Llama-3.
  • Line 10-12: Create a models/ directory if it doesn't exist. This is where the model file will be stored.
  • Line 14-20: Use hf_hub_download() to download the model. The function handles caching automatically.
  • Line 27-28: Standard Python idiom to run the function when the script is executed directly.

Chapter 2: The Agent (agent.py)

This is the brain of the chatbot. It loads the local Llama-3 model and implements the core chat logic with tool calling.

    # n_gpu_layers=-1 attempts to offload all layers to GPU if available

        model_path=MODEL_PATH,
        n_ctx=4096,
        n_gpu_layers=-1, 
        verbose=False
    )
    print("Model loaded successfully!")

    if "weather" in query_lower:
        return mock_results["weather"]
    elif "stock" in query_lower or "price" in query_lower:
        return mock_results["stock"]

    else:

agent.py - Calculator Function
def execute_calculator(expression: str) -> str:
    """Safely evaluates mathematical expressions."""
    try:
        expr = expression.strip()
        # Handle "X% of Y"
        percent_match = re.match(r'(\d+(?:\.\d+)?)\s*%\s*of\s*(\d+(?:\.\d+)?)', expr, re.IGNORECASE)
        if percent_match:
            percent = float(percent_match.group(1))
            value = float(percent_match.group(2))
            return f"{percent}% of {value} = {(percent/100)*value}"
        
        expr = expr.replace('^', '**').replace('×', '*').replace('÷', '/')
        safe_dict = {
            'sqrt': math.sqrt, 'sin': math.sin, 'cos': math.cos, 'tan': math.tan,
            'log': math.log10, 'ln': math.log, 'exp': math.exp, 'abs': abs,
            'pow': pow, 'pi': math.pi, 'e': math.e
        }
        result = eval(expr, {"__builtins__": {}}, safe_dict)
        return f"{expression} = {result}"
    except Exception as e:
        return f"Error calculating '{expression}': {str(e)}"

🔍 Security Note on eval()

  • Line 6-10: Special handling for "25% of 100" style expressions using regex.
  • Line 12: Convert common math symbols (^, ×, ÷) to Python equivalents (**, *, /).
  • Lines 13-17: Create a "safe" dictionary of allowed functions. This prevents arbitrary code execution.
  • Line 18: eval() is used with {"__builtins__": {}} to disable all built-in Python functions, making it safe.
agent.py - Tool Dispatcher
def execute_tool(tool_name: str, tool_input: dict) -> str:
    if tool_name == "web_search":
        return execute_web_search(tool_input.get("query", ""))
    elif tool_name == "calculator":
        return execute_calculator(tool_input.get("expression", ""))
    else:
        return f"Unknown tool: {tool_name}"

🔍 Dispatcher Pattern

  • This function routes tool calls to the appropriate handler.
  • Using .get("key", "") provides a default empty string if the key is missing.
  • Returns an error message for unknown tools instead of crashing.

2.4 The Chat Function (Core Logic)

agent.py - Chat Function
def chat(user_message: str, conversation_history: list = None) -> dict:
    if conversation_history is None:
        conversation_history = []
    
    # Check if model is loaded
    if llm is None:
        return {
            "response": "Error: Local model not loaded. Please check server logs.",
            "tool_calls": [],
            "conversation_history": conversation_history
        }

    # Add user message
    conversation_history.append({"role": "user", "content": user_message})
    
    # System prompt to guide the model to use tools via JSON
    system_prompt = """You are a QA Agent with access to tools.
You MUST use the 'web_search' tool for ANY question about current events, weather, stock prices, or real-time information.
You MUST use the 'calculator' tool for ANY math problem.

Tools available:
1. web_search: Input: {"query": "string"}
2. calculator: Input: {"expression": "string"}

IMPORTANT: To use a tool, you must output ONLY a valid JSON object. Do not add any other text.
Format:
{
  "tool": "tool_name",
  "input": { ... }
}

Example:
User: What is the weather in London?
Assistant: { "tool": "web_search", "input": { "query": "weather in London" } }

If no tool is needed, just answer normally.
"""
    
    # Prepare messages for Llama
    messages = [{"role": "system", "content": system_prompt}] + conversation_history
    
    # 1. Ask Model
    response = llm.create_chat_completion(
        messages=messages,
        temperature=0.2,
        max_tokens=1024
    )

🔍 Key Design Decisions

  • Line 2-3: Use None as default instead of [] to avoid Python's mutable default argument bug.
  • Lines 17-35: The system prompt is critical. It tells the model:
    • What tools are available
    • WHEN to use each tool (weather = web_search, math = calculator)
    • The exact JSON format for tool calls
    • A concrete example to follow
  • Line 44: temperature=0.2 makes the model more deterministic (less random).
agent.py - Tool Call Detection
    content = response["choices"][0]["message"]["content"]
    tool_calls_made = []
    
    # 2. Check for tool usage (JSON pattern)
    tool_json = None
    try:
        # Look for a JSON block or just try parsing the whole content
        # Clean specific markdown code blocks if present
        clean_content = content.strip()
        if "```json" in clean_content:
            clean_content = clean_content.split("```json")[1].split("```")[0].strip()
        elif "```" in clean_content:
            clean_content = clean_content.split("```")[1].split("```")[0].strip()
            
        if clean_content.startswith("{") and "tool" in clean_content:
            tool_json = json.loads(clean_content)
    except Exception:
        pass # Not a valid JSON tool call

🔍 Parsing Strategy

  • Lines 10-13: Models often wrap JSON in markdown code blocks like ```json ... ```. We strip these.
  • Line 15: Only try to parse as JSON if it starts with { and contains "tool".
  • Line 17-18: If parsing fails, we assume it's a normal text response.
agent.py - Execute Tool and Get Final Response
    # 3. Handle Tool Call
    if tool_json and "tool" in tool_json:
        tool_name = tool_json["tool"]
        tool_input = tool_json.get("input", {})
        
        # Record the tool call
        tool_calls_made.append({
            "tool": tool_name,
            "input": tool_input,
            "output": None # Will fill later
        })
        
        # Execute
        result = execute_tool(tool_name, tool_input)
        tool_calls_made[-1]["output"] = result
        
        # Add assistant's *intent* to use tool to history
        conversation_history.append({"role": "assistant", "content": json.dumps(tool_json)})
        
        # Add tool result to history
        conversation_history.append({
            "role": "function",
            "content": f"Tool Output for {tool_name}: {result}" 
        })
        
        # Ask Model again with result
        messages = [{"role": "system", "content": system_prompt}] + conversation_history
        response_2 = llm.create_chat_completion(
            messages=messages,
            temperature=0.7,
            max_tokens=1024
        )
        final_response = response_2["choices"][0]["message"]["content"]
        
        conversation_history.append({"role": "assistant", "content": final_response})
        
    else:
        # No tool used
        final_response = content
        conversation_history.append({"role": "assistant", "content": final_response})
    
    return {
        "response": final_response,
        "tool_calls": tool_calls_made,
        "conversation_history": conversation_history
    }

🔍 The Tool-Calling Loop

  • Lines 7-11: Record the tool call for transparency (shown in the UI).
  • Line 14: Execute the actual tool function.
  • Lines 17-24: Add both the tool call intent AND the tool result to conversation history. This gives the model context.
  • Lines 27-33: Call the model AGAIN with the tool result. temperature=0.7 is higher here for more natural responses.
  • Lines 42-46: Return a clean dictionary with the response, tool calls, and updated history.

3.2 Pydantic Models

server.py - Request/Response Models
class ChatRequest(BaseModel):
    """
    Request body for the /chat endpoint.
    
    - message: The user's question or input
    - session_id: Optional ID to maintain conversation context
    """
    message: str
    session_id: Optional[str] = None


class ChatResponse(BaseModel):
    """
    Response from the /chat endpoint.
    
    - response: The chatbot's answer
    - tool_calls: List of tools that were used (for transparency)
    - session_id: The session ID for this conversation
    """
    response: str
    tool_calls: list
    session_id: str


class EvaluationResponse(BaseModel):
    """
    Response from the /evaluate endpoint.
    
    - results: List of individual test results
    - summary: Aggregate metrics (accuracy, avg score, etc.)
    """
    results: list
    summary: dict


# In-memory session storage
sessions = {}

🔍 Why Pydantic?

  • Pydantic models provide automatic validation. If someone sends a request without a "message" field, FastAPI returns a helpful error.
  • Optional[str] = None means the field is optional with a default of None.
  • sessions dict: Stores conversation history in memory. In production, use Redis or a database.

3.3 API Endpoints

server.py - Chat Endpoint
@app.get("/")
async def root():
    """Serve the frontend HTML file."""
    return FileResponse(os.path.join(os.path.dirname(__file__), "index.html"))


@app.post("/chat", response_model=ChatResponse)
async def chat_endpoint(request: ChatRequest):
    """
    Main chat endpoint - send a message, get a response.
    """
    try:
        # Generate or use existing session ID
        session_id = request.session_id or f"session_{len(sessions)}"
        
        # Get existing conversation history or start fresh
        conversation_history = sessions.get(session_id, [])
        
        # Call the agent
        result = chat(request.message, conversation_history)
        
        # Store updated conversation history
        sessions[session_id] = result["conversation_history"]
        
        return ChatResponse(
            response=result["response"],
            tool_calls=result["tool_calls"],
            session_id=session_id
        )
    
    except Exception as e:
        print(f"Error in chat endpoint: {e}")
        raise HTTPException(status_code=500, detail=str(e))

🔍 How Sessions Work

  • Line 2-4: The root URL (/) serves the HTML frontend directly.
  • Line 14: If no session_id is provided, generate one like "session_0", "session_1", etc.
  • Line 17: Retrieve existing conversation history or start with empty list.
  • Line 23: Save the updated history back to the session store.
  • This allows multi-turn conversations ("What's the capital of France?" → "Paris" → "What's its population?").

4.2 Judge Function

evaluation.py - LLM Judge
def judge_response(question: str, expected: str, actual: str, tool_calls: list) -> dict:
    """Use local LLM to judge the quality of a response."""
    if llm is None:
        return {
            "correctness": 0, "completeness": 0, "relevance": 0, "overall_score": 0, 
            "explanation": "Judge model not loaded"
        }

    judge_prompt = f"""You are evaluating a chatbot's response to a question.

Question: {question}
Expected Answer: {expected}
Actual Response: {actual}
Tools Used: {json.dumps(tool_calls) if tool_calls else "None"}

Evaluate the response on these criteria:
1. Correctness (1-5): Is the information accurate?
2. Completeness (1-5): Does it fully answer the question?
3. Relevance (1-5): Is the response relevant?

Provide your evaluation in this exact JSON format:
{{
    "correctness": ,
    "completeness": ,
    "relevance": ,
    "overall_score": ,
    "explanation": ""
}}

Only output the JSON, nothing else."""

    try:
        messages = [
            {"role": "system", "content": "You are a fair and critical judge."},
            {"role": "user", "content": judge_prompt}
        ]
        
        response = llm.create_chat_completion(
            messages=messages,
            temperature=0.1, # Low temperature for consistent scoring
            max_tokens=500
        )
        
        judge_text = response["choices"][0]["message"]["content"]
        
        # Extract JSON
        if "```json" in judge_text:
            judge_text = judge_text.split("```json")[1].split("```")[0]
        elif "```" in judge_text:
            judge_text = judge_text.split("```")[1].split("```")[0]
            
        evaluation = json.loads(judge_text.strip())
        return evaluation
    
    except Exception as e:
        print(f"Error judging response: {e}")
        return {
            "correctness": 3,
            "completeness": 3,
            "relevance": 3,
            "overall_score": 3,
            "explanation": f"Error during evaluation: {str(e)}"
        }

🔍 LLM-as-Judge Approach

  • This is a popular technique for evaluating LLM outputs without human annotators.
  • The judge model sees: the question, expected answer, actual response, and tools used.
  • temperature=0.1: Very low randomness for consistent, reproducible scores.
  • Scores are on a 1-5 scale for Correctness, Completeness, and Relevance.

4.3 Run Evaluation

evaluation.py - Main Evaluation Loop
def run_evaluation() -> dict:
    """Run the full evaluation suite."""
    results = []
    print("Starting evaluation...")
    print(f"Running {len(EVALUATION_DATASET)} test cases\n")
    
    for i, test_case in enumerate(EVALUATION_DATASET):
        print(f"[{i+1}/{len(EVALUATION_DATASET)}] Testing: {test_case['question'][:50]}...")
        
        try:
            # Run the question through the chatbot
            chat_result = chat(test_case["question"], conversation_history=None)
            
            # Have the LLM judge the response
            evaluation = judge_response(
                question=test_case["question"],
                expected=test_case["expected_answer"],
                actual=chat_result["response"],
                tool_calls=chat_result["tool_calls"]
            )
            
            # Check if correct tool was used
            tools_used = [tc["tool"] for tc in chat_result["tool_calls"]]
            expected_tool = test_case.get("requires_tool")
            correct_tool_used = (expected_tool is None or expected_tool in tools_used)
            
            result = {
                "id": test_case["id"],
                "question": test_case["question"],
                "expected_answer": test_case["expected_answer"],
                "actual_response": chat_result["response"],
                "category": test_case["category"],
                "tools_used": tools_used,
                "expected_tool": expected_tool,
                "correct_tool_used": correct_tool_used,
                "scores": evaluation,
                "passed": evaluation["overall_score"] >= 3.5
            }
            
            results.append(result)
            print(f"   Score: {evaluation['overall_score']}/5 - {'PASS' if result['passed'] else 'FAIL'}")
            
        except Exception as e:
            print(f"   ERROR: {e}")
            results.append({
                "id": test_case["id"],
                "question": test_case["question"],
                "error": str(e),
                "passed": False
            })

🔍 Evaluation Flow

  • Line 12: Run each question through the chatbot (fresh conversation each time).
  • Lines 15-20: Have the LLM judge the response quality.
  • Lines 23-25: Check if the expected tool was used (e.g., calculator for math questions).
  • Line 37: Pass threshold is 3.5/5 (70%).
